# 高级统计课程知识图谱学习路径推荐系统运行手册

本手册适用于 Windows PowerShell、Windows Git Bash、Linux 和 macOS 本机演示。系统不依赖 Docker 或公网服务，生产演示由 FastAPI 同时提供 API 和 Vue 静态页面，统一访问地址为 `http://127.0.0.1:8000`。学生、教师和管理员的界面截图与逐步操作见 [用户操作手册](./用户操作手册.md)。

## 1. 环境要求

| 工具 | 建议版本 | 检查命令 |
|---|---:|---|
| 操作系统 | Windows 10/11、主流 Linux 或 macOS | 按系统检查 |
| 命令环境 | PowerShell 5.1+ 或 Bash 4+ | `$PSVersionTable.PSVersion` / `bash --version` |
| Git | 2.40+ | `git --version` |
| Python | 3.13.x，64 位 | `python --version` |
| Node.js | 24.x | `node --version` |
| pnpm | 11.x | `pnpm --version` |
| SQLite CLI | 3.x，可选 | `sqlite3 --version` |

首次安装需要访问 Python 和 pnpm 包仓库。系统运行和演示阶段不需要联网；知识点中的外部学习资源链接只有在点击打开时需要联网。

## 2. 获取项目

```bash
git clone https://github.com/Mesanura/advanced-stats-kg-path-recommender.git
cd advanced-stats-kg-path-recommender
git switch main
```

若已经位于项目目录，只需确认分支和状态：

```powershell
git status
git log -1 --oneline
```

当前增量交付版本应显示 `v1.2.0` 标签；需求闭环基线保留为 `v1.1.0`，原始原型基线为 `v1.0.0`：

```powershell
git describe --tags --exact-match
```

## 3. 首次安装与初始化

在仓库根目录执行：

PowerShell：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\setup.ps1
```

Windows Git Bash、Linux 或 macOS：

```bash
./scripts/setup.sh
```

脚本依次完成以下操作：

1. 创建项目独立虚拟环境 `.venv`。
2. 校验并安装 `backend/requirements.txt` 和 `backend/requirements-dev.txt`。
3. 按 `frontend/pnpm-lock.yaml` 安装前端依赖。
4. 从 `.env.example` 复制本地 `.env`（仅在文件不存在时）。
5. 执行 Alembic 迁移，创建 `backend/data/app.db`。
6. 导入 25 个知识点并生成 50 名学生的固定种子演示数据。
7. 分别计算规则法和 BKT 的 1,250 条掌握度结果。

PowerShell 与 Bash 安装脚本会先离线校验现有 Python 包；仅在依赖缺失时访问包索引，并使用有限的超时和重试。两者都不会强制升级 pip。需要指定 Python 可执行文件时，可在 PowerShell 中设置 `$env:PYTHON_BIN`，或在 Bash 中设置 `PYTHON_BIN`。

安装完成后，数据基线应为：1 个年级、2 个班级、50 名学生、25 个知识点、2,475 条访问、1,868 条练习和 876 条视频进度记录。

### 3.1 本地配置

默认 `.env`：

```dotenv
APP_ENV=development
DATABASE_URL=sqlite:///./backend/data/app.db
JWT_SECRET=replace-with-a-random-secret
JWT_EXPIRE_MINUTES=480
FRONTEND_ORIGIN=http://127.0.0.1:5173
```

`.env` 已被 Git 忽略。正式演示前建议生成随机 JWT 密钥并替换 `JWT_SECRET`。PowerShell：

```powershell
[Convert]::ToHexString([Security.Cryptography.RandomNumberGenerator]::GetBytes(32))
```

Bash：

```bash
openssl rand -hex 32
```

不要把生成的密钥、`.env` 或运行数据库提交到仓库。

## 4. 启动系统

### 4.1 单地址演示模式

PowerShell：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\run.ps1
```

Bash：

```bash
./scripts/run.sh
```

脚本先构建 Vue 前端，再启动 FastAPI。看到以下信息后访问：

- 系统首页：<http://127.0.0.1:8000>
- OpenAPI 文档：<http://127.0.0.1:8000/docs>
- 健康检查：<http://127.0.0.1:8000/api/v1/health>

在运行脚本所在终端按 `Ctrl+C` 停止服务。

### 4.2 前后端开发模式

PowerShell：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\dev.ps1
```

Bash：

```bash
./scripts/dev.sh
```

开发地址：

- 前端 Vite：<http://127.0.0.1:5173>
- 后端 API：<http://127.0.0.1:8000>

Vite 会把 `/api` 代理到 FastAPI。开发模式支持前后端热更新，适合修改代码；任一服务退出时，脚本会停止另一服务及其子进程。课程演示应使用单地址模式。

## 6. 三角色操作流程

本节用于运维人员快速演示。面向最终用户的完整图文流程、权限边界、数据影响和操作检查清单见 [用户操作手册](./用户操作手册.md)。

### 6.1 学生端

1. 使用任一学号登录，进入“个人学习中心”。
2. 查看综合掌握度、五个能力维度、薄弱知识点、建议学习方向和 25 个知识点热力图；没有建议时界面会显示明确空状态。
3. 在右上角选择目标知识点，单击“生成路径”。
4. 在“当前学习路径”中按“阶段 N/M”查看节点；序号在所有阶段全局连续，目标节点始终位于最后。
5. 单击路径节点打开详情抽屉，查看难度、前置知识和学习资源。
6. 在抽屉中记录访问时长、更新视频进度或提交练习正确/错误。
7. 提交当前路径节点练习后，系统按原目标自动生成并展示更新路径；记录访问时长或视频进度后，旧路径会标记为过期，需要重新生成。

系统把目标的多个前置关系解释为全部必修，并覆盖掌握度低于阈值的全部祖先。默认每阶段最多 8 个节点；依赖超过上限时显示“完整必修依赖已拆分”并均衡分阶段。浅层目标显示 `shallow_target` 对应说明；已掌握目标显示 `target_mastered` 单节点复习说明；超长依赖显示 `staged_dependency` 说明。

### 6.2 教师端

教师端顶部包含四个分区：

- **学情概览：** 选择负责班级或所在年级，切换 BKT/规则法，查看学生数、平均掌握度、五维雷达图、薄弱 TOP5 和关注学生。
- **学生诊断：** 按姓名或学号检索，仅能查看负责班级的个人信息；单击“详情”打开右侧诊断抽屉后，可查看薄弱点、中文建议学习方向、知识点诊断，并代生成分阶段路径。
- **知识图谱：** 使用“知识点 / 前置关系 / 关系图”三个视图；可按章节、难度、名称/编码组合筛选知识点，完成知识点及关系的新增、编辑、删除。系统会拒绝自环、重复边和成环关系，关系变更以事务提交。
- **推荐策略：** 切换默认诊断算法，配置最短计划节点数、每阶段最多节点、阈值和薄弱覆盖、前置衔接、阶段数惩罚、难度跳跃四项评分权重。权重总和必须为 1，薄弱阈值必须小于掌握阈值。

教师可以查看所在年级的匿名聚合结果，但不能读取未负责班级的学生个人诊断；越权请求返回 403。

### 6.3 管理员端

1. 使用管理员账号进入管理端，顶部可切换“用户与班级 / 知识图谱 / 推荐策略”。
2. 在“用户与班级”中使用姓名/用户名和角色筛选账号，通过“年级 / 班级”创建教学组织。
3. 通过“新增账号”创建学生、教师或管理员账号；创建学生时必须填写学号和班级，创建教师时可分配一个或多个负责班级。
4. 表格会直接列出教师负责的具体班级；单击“详情”可查看和修改账号资料、学生班级或教师负责班级，并可重置密码或删除用户。
5. 账号列表可直接启停非当前管理员账号；系统禁止当前管理员停用或删除自己。
5. 在“知识图谱”中使用与教师相同的三个视图和知识点/关系完整 CRUD。
6. 在“推荐策略”中修改诊断算法、分阶段长度、阈值和四项权重；保存后所有当前路径会标记为待更新。

停用账号后该用户不能登录。密码重置会恢复为角色对应的演示默认密码。删除学生账号会级联清理该学生的行为、诊断和路径数据，执行前应确认无需保留。

## 7. 数据库与演示数据

### 7.1 执行迁移

PowerShell：

```powershell
Set-Location backend
..\.venv\Scripts\python.exe -m alembic upgrade head
Set-Location ..
```

Linux/macOS Bash：

```bash
cd backend
../.venv/bin/python -m alembic upgrade head
cd ..
```

Windows Git Bash 将上述 Python 路径替换为 `../.venv/Scripts/python.exe`。

`v1.1.0` 的 Alembic 头版本为 `0002`。该迁移向 `learning_path_items` 增加非空 `stage` 字段并把旧记录回填为 `1`，同时把旧算法生成的 `CURRENT` 路径标记为 `STALE`，用户需要重新生成。接口仍保留扁平 `nodes`，并新增每个节点的 `stage` 和路径级 `stage_count`，旧调用方不需要改为嵌套阶段结构。

### 7.2 重置固定种子数据

重置会删除本地运行数据库中的学习行为、诊断、路径、账号和教学组织，再恢复演示基线。项目源码不会受影响。

PowerShell：

```powershell
Set-Location backend
..\.venv\Scripts\python.exe -m app.cli seed --reset
..\.venv\Scripts\python.exe -m app.cli diagnose --algorithm rule
..\.venv\Scripts\python.exe -m app.cli diagnose --algorithm bkt
Set-Location ..
```

Linux/macOS Bash：

```bash
cd backend
../.venv/bin/python -m app.cli seed --reset
../.venv/bin/python -m app.cli diagnose --algorithm rule
../.venv/bin/python -m app.cli diagnose --algorithm bkt
cd ..
```

Windows Git Bash 将上述 Python 路径替换为 `../.venv/Scripts/python.exe`。

知识图谱的可审查源数据位于 `data/seed/knowledge_graph.yaml`，模拟器导出的复现数据位于 `data/seed/generated/`。

## 10. 常见故障排查

### 10.1 PowerShell 禁止执行脚本

使用带执行策略的命令：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\setup.ps1
```

无需修改系统的全局执行策略。

### 10.2 `python`、`node` 或 `pnpm` 找不到

重新打开终端，分别运行版本检查命令。若仍找不到，将对应安装目录加入用户 `PATH`。pnpm 可通过 Corepack 启用：

```powershell
corepack enable
corepack prepare pnpm@11.7.0 --activate
```

Bash 脚本会依次查找 `pnpm` 和 `pnpm.cmd`，因此可直接在 Windows Git Bash 中使用 pnpm 的 Windows 安装。Linux/macOS 若仍找不到 pnpm，请确认 Corepack 的安装目录已加入 `PATH`。

### 10.3 Bash 提示权限不足或脚本无法执行

先在仓库根目录补充执行权限：

```bash
chmod +x scripts/*.sh
```

也可不修改权限，直接执行 `bash scripts/setup.sh`。四个 Bash 脚本会补齐标准工具目录，并自动识别 Linux/macOS 的 `.venv/bin/python` 和 Windows Git Bash 的 `.venv/Scripts/python.exe`，因此也支持从 PowerShell 直接调用 Git Bash 可执行文件。

### 10.4 8000 或 5173 端口被占用

```powershell
Get-NetTCPConnection -LocalPort 8000,5173 -ErrorAction SilentlyContinue |
  Select-Object LocalPort, State, OwningProcess
```

先停止占用端口的本项目旧进程，再重新运行脚本。不要直接结束无法确认归属的系统进程。

### 10.5 数据库缺表或数据异常

先停止服务，执行第 7.2 节的固定种子重置命令。若只缺迁移，执行 `alembic upgrade head` 即可。

### 10.6 登录后立即返回登录页

确认使用 `127.0.0.1` 而不是在 `localhost` 与 `127.0.0.1` 之间切换；清除该地址的 Cookie 后重试。检查 `.env` 中 `JWT_SECRET` 非空，且系统时间正确。

### 10.7 前端页面空白或图表未显示

```powershell
Set-Location frontend
pnpm install --frozen-lockfile
pnpm build
Set-Location ..
```

随后重新运行 `scripts/run.ps1` 或 `scripts/run.sh` 并强制刷新浏览器。生产模式必须先生成 `frontend/dist`。

### 10.8 Playwright 提示浏览器不存在

```powershell
Set-Location frontend
pnpm exec playwright install chromium
Set-Location ..
```

安装浏览器运行时只需执行一次；它不会进入 Git 仓库。

### 10.9 依赖安装失败

确认网络和代理配置，保留 `pnpm-lock.yaml`，不要手工修改锁文件。清理问题前先记录完整错误信息；恢复后重新运行 `scripts/setup.ps1` 或 `scripts/setup.sh`，脚本可重复执行。
