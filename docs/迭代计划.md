# 迭代计划与验收记录

本文按版本记录目标、状态和可复核证据。功能只有在对应自动化测试、构建和视觉验收通过后才标记为“已验收”。

## v1.2.0 - 操作闭环增强

**状态：** 已验收

**目标：** 在 `v1.1.0` 需求闭环基础上完善管理员账号治理，并让学生提交路径节点练习后自动获得同一目标的更新路径。

| 验收域 | 交付内容 | 状态 | 验收证据 |
|---|---|---|---|
| 学习反馈闭环 | 练习提交后重算掌握度，旧路径转为 `stale`，按原目标生成并直接展示新路径；支持连续提交 | 已验收 | 后端连续提交测试、学生组件测试、桌面与移动 Playwright 点击流程 |
| 管理员账号治理 | 用户详情抽屉；用户名、姓名、学号、学生班级、教师负责班级和状态编辑；账号删除 | 已验收 | 后端权限与级联删除测试、管理员组件测试 |
| 管理安全 | 当前管理员不可停用或删除自身；教师列表直接显示具体负责班级 | 已验收 | API 回归测试和前端组件断言 |
| 使用文档 | 三角色图文操作手册，覆盖权限、主要流程、数据影响和常见问题 | 已验收 | `docs/用户操作手册.md` 与 9 张实际界面截图 |
| 质量门槛 | 后端 >=85%；前端语句/行 >=90%、分支 >=85%、函数 >=70%；构建与双视口 E2E 通过 | 已验收 | 后端 50 项、覆盖率 88.53%；前端 24 项、语句/行 95.83%、分支 87.66%、函数 76.82%；Playwright 8 项 |

### 实测证据

- 后端：50 项 Pytest 全部通过，覆盖率 88.53%。
- 前端：10 个测试文件、24 项 Vitest 全部通过；语句/行 95.83%、分支 87.66%、函数 76.82%，生产构建通过。
- 浏览器：三角色流程在 `1440x900` 与 `390x844` 下共 8 项 Playwright 全部通过；学生流程实际点击提交练习并断言原目标路径自动更新，管理员流程覆盖多班级教师账号治理。

| 文档与发布 | README、运行手册、算法/API/迁移说明、CHANGELOG、1.1.0 版本与验收截图 | 已验收 | README、运行手册、CHANGELOG 与版本号已更新；桌面和移动端验收截图已归档 |

### 实测证据

- 后端：49 项 Pytest 全部通过，覆盖率 87.35%（门槛 85%），无测试失败或资源警告。
- 前端：10 个测试文件、23 项 Vitest 全部通过；语句 98.10%、分支 89.79%、函数 77.77%、行 98.10%。
- 浏览器：学生、教师、管理员 3 条流程在桌面和移动两个项目运行，共 6 项 Playwright 全部通过；断言控制台/页面错误为空、文档无横向溢出、图表画布存在有效像素。
- 性能：规则法全量诊断 123.518ms，BKT 全量诊断 118.129ms，仪表盘 P95 10.820ms，路径生成 5.748ms。
- 迁移：空库可从 Alembic base 升至 `0002`；现有 1.0.3 结构升级后 `stage` 非空、旧当前路径为 `stale`；演示库最终保持 25 个知识点且无 `e2e_` 临时数据。
